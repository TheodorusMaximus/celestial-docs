## System Architecture

### High-Level Design

PipelineVision employs a modular, edge-first architecture optimized for real-time threat detection during aerial pipeline inspection flights.

```
┌─────────────────────────────────────────────────┐
│            Aircraft Pod System                   │
├─────────────────────────────────────────────────┤
│  Sony ILX-LR1 Camera → Video Capture Module     │
│            ↓                                     │
│  NVIDIA Jetson AGX Orin Edge Computer           │
│  ├─ Frame Processing Pipeline                   │
│  ├─ YOLOv8 Inference Engine                     │
│  ├─ GPS/KMZ Validation Module                   │
│  └─ Alert Generation System                     │
│            ↓                                     │
│  iPad Interface (Operator Alerts)               │
└─────────────────────────────────────────────────┘
```

### Core Technology Stack

#### Computer Vision Pipeline
- **Model Architecture**: YOLOv8 (You Only Look Once v8)
  - Pre-trained on COCO dataset
  - Fine-tuned on DOTA aerial imagery (11,268 images)
  - Enhanced with AIDCON construction equipment (9,563 objects)
- **Performance Metrics**: 93.8% mAP@0.5 on aerial vehicle detection
- **Inference Framework**: PyTorch with TensorRT optimization

#### Edge Computing Platform
- **Hardware**: NVIDIA Jetson AGX Orin
  - 275 TOPS AI performance
  - 32GB/64GB LPDDR5 memory
  - Power envelope: 15W-60W configurable
- **Operating System**: JetPack 5.x (Ubuntu-based)
- **Container Runtime**: Docker with NVIDIA Container Toolkit

#### Video Processing
- **Input Source**: Sony ILX-LR1 Industrial Camera
  - Resolution: 12MP (4096 x 3000)
  - Frame Rate: 30-60 FPS capture
  - Interface: USB3 Vision / GigE Vision
- **Processing Pipeline**:
  - Frame capture and buffering
  - Resolution optimization (1920x1080 for inference)
  - Color space normalization
  - Rolling window detection

## Data Architecture

### Training Data Strategy

#### Multi-Tier Approach
1. **Foundation Layer**: DOTA Dataset
   - 11,268 aerial images
   - 15 object categories
   - Provides aerial perspective baseline

2. **Domain Layer**: AIDCON Dataset
   - 9,563 construction equipment objects
   - Ground-level imagery for domain transfer
   - Excavator-specific annotations

3. **Custom Layer**: VanGuard Collection
   - Pipeline corridor specific imagery
   - Operator-validated threat examples
   - Continuous improvement pipeline

### Model Optimization

#### Proxy Detection Strategy
- Use "truck" class as proxy for excavator detection
- Validate with ≥70% correlation threshold
- Progressive refinement with custom data

#### Performance Targets
- **Inference Speed**: Over 10 FPS sustained
- **Latency**: Under 100ms per frame
- **Memory Usage**: Under 4GB GPU memory
- **Model Size**: Under 200MB optimized

- **Infrastructure**: Jetson AGX Orin on pod
- **Data Source**: Sony ILX-LR1 live feed

- **Timeline**: Months 4-7

## Performance Specifications

### Real-Time Requirements
- **Frame Processing**: 10-15 FPS minimum
- **Detection Latency**: Under 100ms end-to-end
- **Alert Generation**: Under 500ms from detection
- **System Uptime**: Over 99% during flights

### Accuracy Targets
- **Recall (Sensitivity)**: Over 90% for critical threats
- **Precision**: Over 80% to minimize false positives
- **Actionable Intelligence Rate**: Over 85% operator confirmation
- **F1 Score**: Over 0.85 for primary threat classes

### Resource Constraints
- **Power Budget**: 60W maximum on pod
- **Thermal Envelope**: -20°C to 50°C operation
- **Vibration Tolerance**: MIL-STD-810G compliance
- **Storage**: 512GB NVMe for model and logs

## Integration Specifications

### VanGuard System Interfaces

#### Camera Integration
- Protocol: USB3 Vision / GigE Vision
- Resolution: 4096x3000 native, 1920x1080 processed
- Frame Rate: 30 FPS capture, 10-15 FPS inference
- Synchronization: GPS timestamp alignment

#### iPad Interface
- Communication: WebSocket/REST API
- Data Format: JSON with base64 image regions
- Update Rate: 1-2 Hz for alerts
- Feedback Loop: Operator confirm/dismiss actions

#### GPS/KMZ Integration
- Coordinate System: WGS84
- Corridor Validation: ±100m from centerline
- Geofencing: Configurable threat zones
- Logging: Timestamped JSON with location
